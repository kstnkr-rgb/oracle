## 2. Назначение и цели

### 2.1 Описание продукта

«Оракул» — развлекательный сервис внутри приложения лояльности «Клуб Друзей», предоставляющий пользователям персональные прогнозы и советы через различные механики: астрологию, таро, нумерологию, руны, советы Оракула. Сервис реализуется как веб-приложение, встраиваемое в МП через WebView.

### 2.2 Бизнес-цели

- Увеличение частоты заходов в МП за счёт ежедневных бесплатных механик.
- Утилизация кешбэк-валюты (БКД) через платный доступ к расширенным функциям.
- Формирование привычки регулярного взаимодействия с приложением.
- Стимулирование покупок в сети Спар через начисление кристалликов.

### 2.3 Целевая аудитория

Авторизованные пользователи МП «Клуб Друзей». Приоритетная аудитория: женщины 32–42 лет, а также молодёжь поколения Gen Z, интересующиеся астрологией и эзотерикой.

## 3. Общие технические требования

### 3.1 Платформа

Веб-приложение, встраиваемое в МП через WebView. Вёрстка адаптивна под экраны iOS и Android всех актуальных размеров. Максимальная ширина контента — 480px.

**Текущее состояние:** Node.js HTTP-сервер (`server.js`) + SPA (`index.html`). Деплой на Railway через GitHub auto-deploy.

### 3.2 Авторизация и доступ

Сервис доступен только авторизованным пользователям МП «Клуб Друзей». При открытии WebView МП передаёт идентификационные данные пользователя. Без авторизации сервис не запускается.

**В прототипе не реализовано** — интеграция будет добавлена при подключении к WebView МП.

## 5. Профиль пользователя

### 5.1 Данные профиля

| Поле | Назначение | Обязательность |
|---|---|---|
| Дата рождения | Знак зодиака, натальная карта, нумерология | Да |
| Время рождения | Натальная карта, точный астрологический расчёт | Да |
| Город рождения | Натальная карта, определение часового пояса и координат | Да |

Хранится в `localStorage` (ключ `oracul_profile`): `{ date, time, city, lat, lon, countryCode }`.

### 5.2 Онбординг

При первом запуске показывается экран онбординга: пользователь вводит дату и время рождения (барабанный скроллер) и город (поле с автодополнением). После сохранения — переход к главному экрану.

Согласие на использование данных реализовано неявно: под кнопкой «К прогнозам» размещён текст «Нажимая «К прогнозам» ты соглашаешься с Правилами акции» со ссылкой.

### 5.3 Ввод города

Поле работает с автодополнением через геокодер. После ввода трёх и более символов появляется список подсказок (название города + страна). Пользователь обязан выбрать город из списка — свободный ввод не принимается, так как сервису нужны точные координаты.

Используемый сервис: OpenStreetMap Nominatim (`featuretype=settlement`). Дедупликация по ключу `name|country`. Координаты сохраняются в профиль при выборе.

### 5.4 Инвалидация прогнозов при изменении профиля

При изменении даты, времени или города рождения все блокировки на повторное получение прогнозов снимаются. Механизм: `_profileHash()` = конкатенация `date|time|city`; при несовпадении хеша блокировка считается недействительной.

### 5.5 Сброс профиля

На экране профиля — кнопка «Сбросить профиль». Очищает данные рождения, все блокировки прогнозов (`forecastLock`), кеш астроданных (`_astroCache`) и удаляет ключ `oracul_profile` из `localStorage`.

### 6.2 Требования к контенту

- Все прогнозы на русском языке.
- Тон: поддерживающий, мудрый, дружелюбный. Риски — мягко и конструктивно.
- Без астрологического жаргона: не использовать «транзит», «натальный», «аспект», «конъюнкция», «трин», «оппозиция».
- Длина: 150–200 слов для расширенных форматов, 20–50 слов для кратких.
- Не использовать длинное тире (—), только короткое (–) или дефис (-).

### 6.3 Детальное описание механик

#### 6.3.1 Гороскоп

Астрологический прогноз на основе натальной карты и актуального положения планет. Доступные горизонты: Сегодня, Завтра, Неделя, Месяц, Год.

Результат: изображение знака зодиака + текст по структуре: общий прогноз (2–3 предл.) + блоки «Работа», «Здоровье», «Любовь» (1–2 предл. каждый).

Блокировка повторных запросов: каждый горизонт доступен один раз за соответствующий период.

| Период | Блокировка действует |
|---|---|
| Сегодня | До конца текущего календарного дня |
| Завтра | До конца следующего календарного дня |
| Неделя | До конца текущей ISO-недели |
| Месяц | До конца текущего календарного месяца |
| Год | До конца текущего календарного года |

При попытке повторного запроса — информационное сообщение с временем следующего доступного прогноза. Кристаллики не списываются. Если профиль изменился — блокировки снимаются.

Прогнозы сохраняются в истории.

ИИ-модель: `claude-haiku-4-5-20251001`. System prompt: `HOROSCOPE_SYSTEM_PROMPT`.

#### 6.3.2 Таро

Два формата:

**Карта дня** — одна карта, выбранная на основе данных рождения и текущей даты. Отображается изображение карты (аркан, масть, стихия, положение) и текстовая интерпретация (3–4 предложения). Предоставляется бесплатно при первом использовании, затем раз в день.

**Расклад на 3 карты** — три карты в позициях «Прошлое», «Настоящее», «Будущее». Три изображения + сводная интерпретация по структуре: Прошлое, Настоящее, Будущее, Общий вывод.

**Не реализовано:** после выбора формата должен показываться экран с картами рубашкой кверху — пользователь самостоятельно выбирает нужное количество карт.

ИИ-модель: `claude-haiku-4-5-20251001`. System prompt: `TAROT_SYSTEM_PROMPT`.

#### 6.3.3 Совет Оракула

Персонализированный совет на основе натальной карты и текущих астрологических транзитов. Пользователь вводит вопрос в свободной форме (до 200 символов). После достижения лимита ввод блокируется, счётчик выделяется красным.

После нажатия кнопки «Получить совет» система проверяет баланс и показывает экран подтверждения: «Вы хотите отправить текст? Изменить его будет нельзя» с двумя кнопками — «Получить совет · 30 💎» и «Изменить сообщение». При нажатии «Изменить сообщение» пользователь возвращается к редактированию.

Ответ (~150–200 слов) отображается в том же окне под полем ввода. После получения — кнопка «Поделиться».

ИИ-модель: `claude-sonnet-4-6`. System prompt: `ORACLE_SYSTEM_PROMPT`.

#### 6.3.4 Толкователь снов

Пользователь описывает сон в свободной форме (до 500 символов). После нажатия «Истолковать» система проверяет баланс и показывает экран подтверждения: «Вы хотите отправить текст? Изменить его будет нельзя» с двумя кнопками — «Истолковать · 30 💎» и «Изменить сообщение». При нажатии «Изменить сообщение» пользователь возвращается к редактированию.

ИИ-модель формирует толкование (~150–200 слов) в позитивном психологическом ключе, от второго лица. После получения — кнопка «Поделиться».

ИИ-модель: `claude-sonnet-4-6`. System prompt: `DREAM_SYSTEM_PROMPT`.

На мобильных устройствах при появлении клавиатуры модальное окно автоматически поднимается через `visualViewport` API.

#### 6.3.5 Нумерология

Персонализированный анализ на основе даты рождения. Рассчитываются: жизненный путь, число судьбы, зов души, число личности, число дня рождения, число личного года, число зрелости.

Результат: сводная сетка чисел с краткой темой каждого + детальная интерпретация + пиковые периоды жизни (с указанием текущего активного) + актуальные жизненные вызовы.

Все тексты переводятся с английского на русский через `TRANSLATE_SYSTEM_PROMPT`.

#### 6.3.6 Руны

Три формата гадания на скандинавских рунах:

- **Одна руна** — ответ на конкретный вопрос или общий совет.
- **Три руны** — расклад «Прошлое», «Настоящее», «Будущее».
- **Руна ситуации** — анализ текущего положения дел.

Для каждой руны: символ, имя, ключевые слова, стихия, положение (прямое или перевёрнутое), текстовая интерпретация. В конце расклада — общий итог и совет.

#### 6.3.7 Магический шар

Пользователь нажимает кнопку и получает один из четырёх случайных ответов: «Да», «Нет», «Не знаю», «Спроси позже». Не требует данных рождения, не обращается к внешним API. История не сохраняется (`noHistory: true`).

#### 6.3.8 Печенье с предсказанием

Пользователь «разламывает» виртуальное печенье — изображение меняется на разломанное, появляется короткое позитивное предсказание (до 100 символов), сгенерированное ИИ. Не требует данных рождения. История не сохраняется (`noHistory: true`).

ИИ-модель: `claude-haiku-4-5-20251001`. System prompt: `COOKIE_SYSTEM_PROMPT`.

## 7. Игровая валюта — Кристаллики

### 7.1 Стартовый баланс

При первом запуске начисляется: 100 кристалликов + 300 бонусов Спар.

### 7.2 Способы получения

- **Покупка за БКД** — обмен бонусов Клуба Друзей по фиксированным пакетам.
- **Покупка за рубли** — через платёжный сервис KD Pay в МП.
- **За покупки в сети Спар:** 1 кристаллик за каждые полные 100 рублей в чеке; +10 кристалликов при оплате через KD Pay; +10 кристалликов при покупке в Spar Online.

**Текущее состояние прототипа:** интеграция с ML и KD Pay не реализована. Кнопки покупки в интерфейсе есть, реальная транзакция не проводится.

### 7.3 Пакеты покупки

| Кристалликов | Бонусы Клуба Друзей (БКД) | Рубли (KD Pay) |
|---|---|---|
| 20 | 50 БКД | 50 ₽ |
| 50 | 100 БКД | 100 ₽ |
| 100 | 180 БКД | 180 ₽ |

### 7.4 Логика списания

Кристаллики списываются **после** успешной генерации результата. Если баланс недостаточен — механика не запускается, показывается попап «Недостаточно кристалликов». Если доступна бесплатная попытка — кристаллики не тратятся.

Все операции фиксируются в истории транзакций (`crystalLedger`), доступной в разделе профиля.

### 7.5 Платёжные механизмы

Оплата исключительно через программу лояльности — KD Pay или обмен БКД. Нативная оплата через App Store / Google Play In-App Purchase не используется. Фактическое проведение платежа выполняется на стороне МП; сервис лишь инициирует запрос и получает подтверждение.

## 8. Интерфейс и пользовательский опыт

### 8.1 Экраны

**Онбординг** — показывается при первом запуске. Ввод даты, времени рождения (барабанный скроллер) и города (автодополнение). Согласие на обработку данных. После сохранения — переход к главному экрану.

**Главный экран (Механики)** — список всех механик. Закреплённая шапка: баланс бонусов Спар и кристалликов, кнопка профиля. При нажатии на механику раскрывается список подтипов (если есть) с ценами.

**Профиль** — данные рождения, история прогнозов по каждой механике, история транзакций кристалликов, кнопка сброса профиля.

**Результат** — модальное окно поверх главного экрана. Содержит заголовок, контент прогноза, кнопку закрытия и кнопку «Поделиться» (кроме Магического шара и Печенья).

### 8.2 Состояние загрузки

При запуске прогноза модальное окно открывается немедленно с анимацией загрузки. Текст подсказки сменяется каждые несколько секунд («Читаю звёзды…», «Составляю прогноз…» и т.д.). Для гороскопа, таро, нумерологии и рун — анимированная обложка.

### 8.3 История прогнозов

Все успешно полученные прогнозы сохраняются и доступны в разделе «Ваши предсказания» в профиле, в обратном хронологическом порядке. Нажатие на запись открывает полный текст без повторного списания кристалликов. Механики с `noHistory: true` (Магический шар, Печенье) в историю не сохраняются.

### 8.4 Поделиться

В окне результата для всех механик кроме Магического шара и Печенья доступна кнопка «Поделиться». Текст прогноза + стандартный текст о сервисе передаётся в системный диалог шеринга МП (`navigator.share`).

### 8.5 Барабанный ввод даты и времени

- Три колонки для даты: год (от 1940 до текущего), месяц (бесконечная прокрутка), день (бесконечная прокрутка).
- Отдельный барабан для времени: часы и минуты (оба с бесконечной прокруткой).
- Поддержка тач-скролла и колёсика мыши.
- Константы: `DRUM_H = 30px`, `DRUM_PAD_PX = 60px`, высота барабана `.drum-picker { height: 150px }`.

### 8.6 Защита от повторных нажатий

Все кнопки запуска механик после нажатия переходят в неактивное состояние на 1 секунду, исключая случайную двойную отправку.

**В прототипе не реализовано.**

### 8.7 Поведение при сворачивании и закрытии

При сворачивании МП пользователь при возврате попадает на тот же экран. Если МП закрыто во время генерации прогноза — прогноз не сохраняется в историю, кристаллики не списываются.

**В прототипе не реализовано.**

## 9. Архитектура и интеграции

### 9.1 Схема работы

1. Пользователь открывает сервис из МП.
2. Выбирает механику и подтип. Система проверяет баланс кристалликов.
3. Бэкенд отправляет запрос к Astro API — получает астрологические данные.
4. Данные передаются в ИИ-модель, которая генерирует текст прогноза на русском языке.
5. Прогноз отображается пользователю. Кристаллики списываются по факту успешного результата.

Бэкенд выступает прокси между фронтендом и внешними API. API-ключи хранятся на сервере, в браузер не передаются.

### 9.2 Внешние API

| Сервис | Назначение | Примечание |
|---|---|---|
| astrology-api.io | Позиции планет, натальная карта, таро, руны, нумерология | Платный API, тарифный лимит |
| Claude (Anthropic) | Генерация текстов прогнозов, перевод | haiku — большинство механик; sonnet — Оракул, Сны |
| OpenStreetMap Nominatim | Геокодирование по названию города | `featuretype=settlement` |
| Manzana Loyalty (ML) | Баланс БКД, история транзакций | Не реализовано в прототипе |
| KD Pay | Покупка кристалликов за рубли | Не реализовано в прототипе |

### 9.3 ИИ-модели и system prompts

| Механика | Модель | System prompt |
|---|---|---|
| Гороскоп | claude-haiku-4-5-20251001 | `HOROSCOPE_SYSTEM_PROMPT` |

### 9.4 Кэширование
